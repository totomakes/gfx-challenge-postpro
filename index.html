01
Pre-edit · Producer / Post-lead

Ingest & proxies

Before anything reaches an editor, the camera negatives are protected and lightweight versions are built for cutting.

  • Back up and catalog all RAW footage — originals are never edited directly.
  • Generate proxies with the Blackmagic Proxy Generator.
  • Send proxies + sound recordings to the editors.

Editors cut against proxies; the XML export later tells us exactly which RAW clips to pull.

02
Offline · Editor

Offline edit

Build the story on the spec timeline. Grantee stills arrive as JPGs and go in as temporary placeholders.

  • Cut on a sequence that matches the Sequence Spec exactly.
  • Drop in grantee JPG placeholders where final images will live — these get replaced after lock.
  • Publish Review versions for feedback rounds.
Deliverable — Review version 4K · H.264 · .mov · 20,000 kbps → Recipe A
03
Picture lock · Editor + Fuji team

Picture lock & final assets

Once the cut is locked, swap placeholders for real assets and request the overlays that ride on top of them.

  • Export an image list of every still used in the cut and send it to the Fuji team.
  • Receive full-resolution / final images + the matching metadata graphics (PNG overlays).
  • Confirm every final image is sRGB. Convert any other color space to sRGB before placing — this keeps colors identical to what was delivered and prevents mismatches.
  • Each metadata PNG sits directly on top of its image, for the full duration that image is on screen.
  • Export an XML so we can extract and archive only the RAW files actually used.
Color management: JPGs were placeholders only. Finals must be sRGB — if they arrive as Adobe RGB, Display P3 or ProPhoto, convert to sRGB (assign-then-convert) before they hit the timeline.
04parallel
Parallel track A · Editor → Sound design

Audio handoff

Right after lock, hand the sound team everything they need to work independently of color.

  • Export an AAF with embedded clips so sound designers get every channel / audio line separated.
  • Export a 1080p H.264 reference video so they can sync to picture.
Deliverable — Sound handoff AAF (embedded) + 1080p H.264 .mp4 · 20,000 kbps → Recipes C & B

You wrote “AAC” — note that the format that carries separated tracks + embedded clips is AAF (Advanced Authoring Format). AAC is a final mixdown codec and can’t hold separate lines, so the handoff is an AAF.

04parallel
Parallel track B · Colorist → Editor

Color

Color grades the locked picture from the conformed RAW and returns a single, edit-ready clip.

  • Colorist receives the conformed files (via the XML / RAW pull).
  • Grade is delivered back as a single ProRes 422 HQ export the editor can lay straight into the timeline.
Deliverable — Color return ProRes 422 HQ · single file · .mov → Recipe E
05
Online · Editor

Online / conform

Bring the three streams back together and render the version stakeholders sign off on.

  • Lay color + final mix + graphics back onto the timeline.
  • Build the full graphics stack — bug, letterbox, metadata overlays, intro & outro.
  • Export the Online version and upload to Frame.io for approval.
Deliverable — Online (approval) 4K · H.264 · .mov · 20,000 kbps → Frame.io → Recipe A
06
Delivery · Editor

Master files & delivery

Once approved, output two masters. Both are ProRes 422 HQ and go to the agreed host.

Master 1

Clean cut

No GFX / VFX over the images across the whole cut, and no subtitles. Keep the intro & outro sequences.

Master 2

Full quality

Everything on — bug, letterbox, metadata overlays, subtitles, intro & outro.

Deliverable — Masters ×2 ProRes 422 HQ · .mov → Frame.io or Dropbox → Recipe E
09 — Layering

The graphics stack

Top layer wins. Build it in this order so nothing covers what it shouldn’t.

  1. TOP
    Metadata graphic (PNG)

    Always directly above its image, for as long as that image is shown. One per still.

  2. ↑
    Fujifilm bug

    Persistent over the whole video — except while full-res images are on screen, and never over the intro.

  3. ↑
    Letterbox

    Maintained consistently across the body of the film.

  4. ↑
    Subtitles

    On for the full-quality master · off for the clean master.

  5. BASE
    Picture (color + edit)

    Graded footage and final images.

Opening

Intro sequence

Plays clean — no bug over the intro.

Closing

Outro + end card

Credits and Fujifilm information. The last frame holds on the end-card text — never cut to black.

13 — Shared language

Glossary

Proxy
Lightweight, low-bitrate copy of the RAW used for fast editing. Conformed back to RAW for finishing.
Picture lock
The edit is final — no more timing or shot changes. Triggers final assets, audio & color.
Bug
The persistent Fujifilm logo over the video (hidden over full-res images and the intro).
Letterbox
The fixed black bars / matte framing the picture throughout the body.
Metadata graphic
PNG overlay riding directly on top of each final image, supplied by the Fuji team.
AAF
Advanced Authoring Format — audio interchange carrying separated tracks with embedded clips for sound design.
Online / Conform
Reassembling color + audio + graphics at full quality for the approval render.
ProRes 422 HQ
High-quality mastering codec used for the color return and both final masters.
sRGB
The required color space for all final images. Convert anything else before placing it.
